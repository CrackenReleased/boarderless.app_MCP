Model Context Protocol Tool Registry & API Specifications

The Boarderless MCP server exposes functional endpoints below. Provide this registry to your LLM when configuring custom harnesses like Ollama, Hermes, or OpenClaw.

⚙️ get_server_status Diagnostic
Description: Returns full diagnostics for the Boarderless MCP server. Call this first to verify browser connectivity, authentication, and bridge state.
Arguments: None ({})
Key Output Payload: 
{ "ready": true, "checks": [ { "check": "browser_port", "passed": true } ] }
📋 get_board_state Canvas Read
Description: Returns the canvas elements as a structured spatial ledger.
Arguments: None ({})
Key Output Payload: 
{ "schema": "boarderless.boardSnapshot.v1", "objects": [ { "id": "rect-1", "objectKind": "shape", "x": 20, "y": 20 } ] }
✍️ mutate_object Canvas Write
Description: Modifies position, size, rotation, fill/stroke, text content, font family, scale, opacity, alignment or corners of an object.
Arguments:
• id: string (Required)
• x / y / width / height / rotation / opacity: number (Optional)
• fill / stroke / text: string (Optional)
📐 calculate_export_bounds Canvas Read
Description: Returns the bounding box encompassing all objects on the active canvas board.
Arguments: None ({})
📁 get/set_board_workspace · export/import_board_file Durable Files
Description: Keeps the canonical schema-v2 .bdrl.json board in the agent's local project directory. Successful canvas mutations autosave atomically; explicit import/export tools support reliable handoff and resumption.
Boundary: Board files must remain inside the configured workspace.
💾 export_board Canvas Action
Description: Exports the current canvas to PNG, PDF, or SVG. SVG and PDF require a Pro plan subscription.
Arguments:
• format: "png" | "pdf" | "svg" (Required)
• mode: "canvas" | "selection" (Optional, defaults to `"canvas"`)
➕ create_object Canvas Write
Description: Create a new object (text or shape: rect, ellipse, triangle, arrow) on the canvas.
Arguments:
• type: "text" | "rect" | "ellipse" | "triangle" | "arrow" (Required)
• x / y / width / height / strokeWidth: number (Optional)
• text / fill / stroke: string (Optional)
❌ delete_objects Canvas Write
Description: Delete one or more objects by their IDs from the canvas.
Arguments:
• ids: string[] (Required)
↩️ history_undo Canvas Action
Description: Undo the last action on the canvas.
Arguments: None ({})
↪️ history_redo Canvas Action
Description: Redo the next action in the history queue on the canvas.
Arguments: None ({})
📦 group_objects Canvas Write
Description: Group multiple canvas objects under a unique groupId.
Arguments:
• ids: string[] (Required)
🔓 ungroup_objects Canvas Write
Description: Ungroup objects belonging to a specific groupId.
Arguments:
• groupId: string (Required)
↕️ reorder_object Canvas Write
Description: Reorder z-index layering of an object (bring to front, send to back, forward, backward).
Arguments:
• id: string (Required)
• action: "front" | "back" | "forward" | "backward" (Required)
📁 graduation_rename_photos File System
Description: Renames local photo files sequentially or by filling gaps. Does not require browser/live session.
Arguments:
• seniorsDir: string (Required absolute path)
• mode: "sequential" | "gap_fill" (Required)
🖼️ graduation_standardize_images File System
Description: Converts progressive JPEGs and HEIC photos to standard RGB JPEGs. Does not require browser/live session.
Arguments:
• seniorsDir: string (Required absolute path)
Security & Client Consent Protocols
🛡️ Localhost Isolation
The Boarderless MCP server attaches to Chrome/Edge using the Chrome DevTools Protocol (CDP) exclusively bound to the 127.0.0.1 local loopback address. No external network clients can connect to or query your canvas state.
✍️ Explicit Interaction Verification
Every mutation request (e.g. creating, moving, deleting, or editing visual objects) triggers a slide-out consent dialog directly inside the Boarderless web app. The server is sandboxed and cannot execute any mutations on your board until you click "Accept" in the UI.
👥 Profile Isolation
By default, the Isolated Profile stores cache and cookies in a separate directories (boarderless-mcp-profile) in your local app data. This ensures your daily browsing cookies, passwords, and sessions remain completely separated from the AI workspace.
🎨 Boarderless Capabilities Index (RAG Index)
This application operates as a bridge for the following core Boarderless capabilities:
  • Canvas & Viewport: Zoom, Pan, alignment grid snap, and z-order sorting.
  • Element Selection & Mutate: Coordinates (x/y), dimensions (width/height), angles, opacities, corner radius, borders, strokes, fills, typography fonts, sizes, and text content alignment.
  • Media & Image Grades: Flip operations, image mask cropping, and filter grades.
  • Local File Operations: Sequentially rename photo directories and standardize baseline color properties for HEIC/JPEG images.
See the full catalog at docs/features_catalog.md inside the MCP application directory.
